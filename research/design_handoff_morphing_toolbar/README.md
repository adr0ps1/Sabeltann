# Handoff: Sabeltann — "Morphing Toolbar" Chrome (Direction 1A)

## Overview
This redesigns Sabeltann's window chrome so the **top bar is a single horizontal toolbar that reshapes its controls per `ContentMode`** (Welcome → Picker → Live TV → Movies → Series → Movie/Series Detail → Playback). It replaces the current `CategoryBar` (Row 1 of `MainWindow.axaml`) and folds the playback transport controls into the top bar. **The bottom transport `Border` (`x:Name="TransportBar"`) and the bottom `StatusBar` are being removed** — do not reimplement them.

The design also restyles the menus and dialogs (File menu, Xtream Login, Settings, Series episodes, Movie detail, Playback + Debug, Update dialog) in the same palette so the whole app is visually consistent.

## About the Design Files
The file in this bundle (`Menu Frames.dc.html`) is a **design reference created in HTML** — an interactive prototype showing intended look and behavior. It is **not production code to copy**. The task is to **recreate these designs in Sabeltann's existing Avalonia 12 / AXAML environment**, using the project's established patterns: `Window.Styles` selectors, `{Binding}` to `MainViewModel`, the existing `ContentMode` enum, converters, and `IsVisible` toggles. Translate the HTML/CSS into AXAML controls and brushes — do not embed HTML.

How to view the reference: open `Menu Frames.dc.html` in a browser.
- **Turn 1 (bottom of page), frame `1A`** is the approved direction — a live, clickable window. Click the nav tabs, open a tile, hit Play, or click the green connection dot to walk through every mode and watch the toolbar morph. (Frame `1B` is the rejected alternative — ignore it.)
- **Turn 2 (top of page), frames `2A`–`2G`** apply 1A's chrome to the remaining viewmodels/dialogs.

## Fidelity
**High-fidelity.** Colors, spacing, radii, and typography are final. Recreate pixel-accurately using Avalonia controls and the tokens below. Sizes in the prototype are in CSS px and map 1:1 to Avalonia device-independent pixels.

Mono accents for posters/avatars are generated by hue: `hsl(H, 42%, 40%)` for channel avatars, `repeating-linear-gradient(135deg, hsl(H,33%,30%) 0 9px, hsl(H,33%,25%) 9px 18px)` for posters, `linear-gradient(180deg, hsl(H,36%,26%), hsl(H,28%,13%))` for detail backdrops.

### Typography
- **UI font**: system UI stack — on Windows this is **Segoe UI Variable / Segoe UI** (Avalonia default `FontFamily`). Don't add a custom font for body.
- **Mono font**: `JetBrains Mono` (weights 500/600) for the `SABELTANN` wordmark, count pills, badges, and the DEBUG label. If not bundled, fall back to `Consolas`.
- Sizes (px → Avalonia `FontSize`): wordmark 10.5–11 (letter-spacing 2.5), screen title 14–15 SemiBold, body 12.5–13, tab labels 12.5, sub labels 11–12, badges 9–11.

### Spacing & shape
- 4px grid. Common gaps: 8, 10, 12, 16, 20.
- Title bar height **40** (main window) / **36** (dialogs). Toolbar height **54**.
- Corner radius: pills/buttons **8**, cards/rows **10–11**, windows **12–13**, logo square **5–6**.
- Toolbar horizontal padding **12**. Card padding **11**.
- Window shadow: `0 30px 70px -30px rgba(0,0,0,.8)`.

### Motion
- Toolbar content change: fade + 5px downward slide-in, **250ms ease** (`barMorph`). Implement with an Avalonia `TransformTransition` + `DoubleTransition` on the swapped panel, or a quick opacity/`RenderTransform` transition when the active sub-bar `IsVisible` flips.
- Content pane change: fade + 8px slide-up, **280ms ease** (`paneIn`).
- Hover on rows/cards: background `Surface → Surface2`, border `Line → #3A3F4A`, ~150ms.

## Interactions & Behavior
- **Mode switching** drives the whole toolbar+content swap. Wire tab clicks to set `Mode` (Live/Movies/Series); `←` back uses the existing `GoBackCommand`; tiles use `SelectChannelCommand` / `SelectShowCommand` / movie-open commands; `▶ Play` triggers playback.
- **Toolbar transitions**: animate the incoming sub-bar (opacity 0→1, translateY -5→0, 250ms). Avalonia: put a `Transitions` collection on each mode panel and toggle `IsVisible`/opacity, or animate via `RenderTransform`.
- **Hover states**: list rows and tiles lighten bg + border (150ms); picker cards get an amber border on hover.
- **Connection dot** reflects `IsConnected` (`Good` when connected); the prototype makes it clickable only to demo the Welcome state.
- **Favorites `★`** is the existing `ShowFavoritesOnly` toggle (amber when active).
- **DEBUG** toggles the bottom-right overlay (`ShowDebugOverlay`), unchanged behavior, new styling.

## State Management
No new state required beyond the existing `MainViewModel`. Reuse: `Mode`/`ContentMode`, `IsWelcome/IsPicker/IsLiveTv/IsMovies/IsSeries/IsMovieDetail`, `ShowVideo`, `IsConnected`, `Categories/SelectedCategory`, `SearchText`, `ShowFavoritesOnly`, `ChannelCount`, `ShowDebugOverlay`, `FilteredChannels`, `VodBrowser/SeriesBrowser/MovieDetail`. Add a `Playback` view-state only if you formalize the playback toolbar (today it keys off `ShowVideo`). Decide where the removed bottom-transport commands (`TogglePlayPause`, `Stop`, `Rewind`, `Forward`, volume, seek) live — see the 2F note.
